Introduction

This guide provides a comprehensive overview of your Vue.js application, highlighting the key technologies, project structure, and important functionalities.

Technologies Used

  • Vue.js 3: Progressive JavaScript framework for building user interfaces.
  • TypeScript: Superset of JavaScript adding static typing.
  • Vuetify: Material Design component framework for Vue.js.
  • Tailwind CSS: Utility-first CSS framework for rapid UI development.
  • Pinia: State management library for Vue.js.
  • Vue Router: Official router for Vue.js.
  • Vue Query: Data-fetching library for managing server state.
  • AWS Amplify: Tools and services for building secure, scalable applications.
  • AWS CDK (Cloud Development Kit): Infrastructure as code tool for defining cloud resources.
  • Chart.js: JavaScript library for data visualization.
  • Vite: Build tool and development server for modern web projects.

Boardwalk Project Structure

The project follows a typical structure for a Vue.js application, organized into components, views, services, and configuration files.

Components

1. AdminNavBar.vue

  • Path: src/components/AdminNavBar.vue
  • Description: Navigation bar tailored for admin users. Dynamically displays navigation items based on user roles, including options like Products, Users, Orders, etc. Conditionally adds a “Migrations” item for developers.

2. AppBar.vue

  • Path: src/components/AppBar.vue
  • Description: The main application bar that includes navigation buttons such as Home, Admin (visible only to admins), Cart, and Profile. Displays the number of items in the cart.

3. ItemCard.vue

  • Path: src/components/ItemCard.vue
  • Description: Displays individual product details, including images, prices, sizes, and stock information. Allows users to select size and quantity, and add items to the cart. Handles custom fields if required.

4. CartItem.vue

  • Path: src/components/CartItem.vue
  • Description: Represents items in the shopping cart. Enables quantity adjustments, viewing product details, and deleting items. Integrates with the cart management services.

5. AuthWrapper.vue

  • Path: src/components/Auth/AuthWrapper.vue
  • Description: Wraps around the application to handle authentication using AWS Amplify. Ensures that routes requiring authentication are protected and redirects unauthorized users accordingly.

Views

Storefront Views

PersonalInfoView.vue
  • Path: src/views/Storefront/PersonalInfoView.vue
  • Description: Collects personal information from users during the checkout process. Validates input fields and navigates to the review page upon successful submission.

Admin Views

AdminReportsView.vue
  • Path: src/views/Admin/AdminReportsView.vue
  • Description: Displays various reports and analytics for administrators. Includes dashboards with charts (using Chart.js), data tables for processing fees, and options to export data as CSV files.

Routing

  • File: src/router/index.ts
  • Description: Configures application routes, including nested routes for admin sections. Implements route guards to restrict access based on user roles (e.g., admin, developer). Handles redirections and ensures that authenticated users can access protected routes.

State Management

Pinia Stores

useAuthStore
  • Path: src/stores/auth.ts (Assumed)
  • Description: Manages authentication state, including user roles and authentication status.
useCheckoutStore
  • Path: src/stores/checkout.ts (Assumed)
  • Description: Handles state related to the checkout process, such as storing user information and selected items.

Services

  • useCart

    • Path: src/services/useCart.ts (Assumed)
    • Description: Provides methods to add, remove, and manage items in the shopping cart.

  • useProducts

    • Path: src/services/useProducts.ts (Assumed)
    • Description: Fetches and manages product data from the backend.

  • useOrders

    • Path: src/services/useOrders.ts (Assumed)
    • Description: Handles fetching and processing order data.

  • useReturns

    • Path: src/services/useReturns.ts (Assumed)
    • Description: Manages return-related data and operations.

  • useUsers

    • Path: src/services/useUsers.ts (Assumed)
    • Description: Fetches and manages user information.

Backend Configuration

AWS Amplify and CDK

  • ImageCloudFront Stack

    • Path: amplify/backend/custom/ImageCloudFront/cdk-stack.ts
    • Description: Defines CloudFront distributions with origin access controls to securely serve images from S3. Configures bucket policies and outputs the CloudFront URL.

Amplify Resources Reference

  • Path: amplify/backend/types/amplify-dependent-resources-ref.d.ts
  • Description: Provides TypeScript definitions for Amplify-dependent resources like APIs, authentication, storage, and functions.

Entry Points

main.ts

  • Path: src/main.ts
  • Description: Initializes the Vue application, integrates plugins like Vuetify, Pinia, Vue Query, and sets up AWS Amplify configurations.

App.vue

  • Path: src/App.vue
  • Description: The root component that incorporates the authentication wrapper and includes the main layout with the AppBar and RouterView.

Common Models

  • models.ts

    • Path: src/common/models.ts
    • Description: Defines TypeScript interfaces and types used across the application, such as CustomFieldValue and EditableFields.

Key Features and Functionalities

1. Authentication and Authorization

  • Utilizes AWS Amplify for user authentication.
  • Protects routes using Vue Router’s navigation guards based on user roles (e.g., adminRequired, devRequired).
  • Displays different navigation options in the AdminNavBar based on user permissions.

2. State Management with Pinia

  • Centralizes application state, making it accessible across components.
  • Manages authentication states, checkout information, and cart items efficiently.

3. Routing and Navigation

  • Organizes routes for both storefront and admin sections.
  • Implements nested routing for admin sub-sections like Products, Users, Orders, etc.
  • Ensures secure access to sensitive routes through route guards.

4. UI Components with Vuetify and Tailwind CSS

  • Leverages Vuetify’s rich set of components to build responsive and aesthetically pleasing interfaces.
  • Uses Tailwind CSS for utility-first styling, allowing rapid UI development.
  • Customizes themes and styles to align with the application’s branding.

5. Data Visualization with Chart.js

  • Integrates Chart.js for rendering dynamic charts in admin reports.
  • Displays sales data and customer frequency through Line and Pie charts.

6. Backend Integration with AWS Amplify

  • Configures GraphQL APIs using AppSync.
  • Manages storage with S3 and serves content securely via CloudFront.
  • Deploys Lambda functions for server-side logic and administrative tasks.

7. State and Data Handling with Vue Query

  • Manages data fetching, caching, and synchronization with server state.
  • Utilizes queries and mutations for interacting with APIs, ensuring efficient data management.

8. Infrastructure as Code with AWS CDK

  • Defines and deploys cloud resources programmatically.
  • Configures secure S3 buckets and CloudFront distributions with origin access controls.

9. Custom Fields and Product Specifications

  • Handles custom fields for products, allowing for dynamic specifications like integers, booleans, and strings.
  • Provides dialogs for editing and entering custom specifications when adding products to the cart.

10. Error Handling and User Feedback

  • Implements snackbars to provide immediate feedback to users on actions like adding items to the cart.
  • Utilizes loading states and disabled buttons to enhance user experience during data operations.

Recaps of Main Technologies Used

Vue.js 3

Vue.js 3 is a progressive JavaScript framework for building user interfaces. Key features include:

  • Composition API: Offers better code organization and reusability.
  • Reactive Data Management: Facilitates reactive programming.
  • Virtual DOM: Ensures efficient rendering.
  • Component-Based Architecture: Promotes modularity.
  • Template Syntax: Uses directives like v-for, v-if.
  • Computed Properties and Watchers: For derived data and side effects.
  • Lifecycle Hooks: Such as onMounted, onUpdated.

Vuetify

Vuetify is a Material Design component framework for Vue.js. Key features include:

  • Responsive Grid System
  • Pre-Built Components: Form controls, navigation components, data display.
  • Customizable Themes and Styles
  • Consistent Material Design Aesthetics

Tailwind CSS

Tailwind CSS is a utility-first CSS framework. Key features include:

  • Utility Classes: For common CSS properties.
  • Responsive Design Utilities
  • Customizable Configuration
  • PurgeCSS Integration: For optimized production builds.

Vue Router

Vue Router is the official routing library for Vue.js applications. Key features include:

  • Nested Route/View Mapping
  • Dynamic Route Matching
  • Navigation Guards: For authentication and authorization.
  • Lazy Loading of Routes

Pinia

Pinia is the official state management library for Vue.js applications. Key features include:

  • Modular Store Design
  • TypeScript Support
  • Devtools Integration
  • Hot Module Replacement
  • Plugins for Extending Functionality

AWS Amplify

AWS Amplify is a set of tools and services for building full-stack applications. Key features include:

  • Authentication (Cognito)
  • API (AppSync, API Gateway)
  • Storage (S3)
  • Functions (Lambda)
  • GraphQL Support
  • CLI for Configuration and Deployment

GraphQL

GraphQL is a query language for APIs. Key features include:

  • Declarative Data Fetching
  • Strong Typing
  • Single Endpoint for All Data Operations
  • Efficient Data Loading
  • Real-Time Updates with Subscriptions

Vite

Vite is a build tool and development server for modern web projects. Key features include:

  • Fast Cold Server Start
  • Instant Hot Module Replacement (HMR)
  • On-Demand Compilation
  • CSS Pre-Processor Support
  • Out-of-the-Box TypeScript Support
